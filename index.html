Jellyfish Documentation

Jellyfish is a bare-bones (kinda like a Jellyfish!) HTML/SCSS framework designed to create flexible grid-based layouts, with only the most essential elements and utilities built in. Jellyfish allows for a solid base for web development work, reducing the bloat and clutter of more commercial and well-known products. Jellyfish is designed for ease-of-use expansion and is a constantly evolving product ready to adapt to any web project.

Framework last updated: July 2018

Directory Structure

Jellyfish uses a fairly straightforward directory structure, detailed below. Partials are used to split the scss into manageable sections.

.brackets.jsonSettings for Adobe Brackets. Can be safely removed if using a different IDE.
/assetsHolds all required assets.
-/css/style.cssCore stylesheet as compiled by base.scss.
-/js/scripts.cssBy default, only contains jQuery required by the navigation menu.
-/scssHolds scss files. Files are listed below in the order that they are @included in base.scss
--/base.scssCompiles all partials into ../css/style.css .
--/_mixins.scss and /mixinsContains mixins to aid in the development of scss files.
--/_settings.scssStores Jellyfish settings including grid settings, breakpoint settings, default font settings, color map and base sizes.
--/_reset.scssContains the fantastic CSS reset by Eric Meyer as well as setting all elements to box-sizing: border-box
--/_global.scssContains scss that modifies elements found across all areas of a website, including typography, links, images and utilities.
--/_grid.scssBuilds the grid system.
--/_nav.scssContains scss that modifies elements found in the main navigation area of a website.
--/header.scssContains scss that modifies elements found in the main header area of a website.
--/_content.scssContains scss that modifies elements found in the main content area of a website.
--/_sidebar.scssContains scss that modifies elements found in the sidebar / widgets area of a website.
--/_footer.scssContains scss that modifies elements found in the main footer area of a website.
/docsHolds files required by the documentation and can be safely removed for live builds.
-/docs.jsjQuery required by the documentation
index.htmlDocumentation. Can be safely removed for live builds.

Basic HTML elements

Typography

Global font families, line height and size are defined in _settings.scss. With the grid breakpoint settings, a new $base-font-size and $base-line-height can be defined for each breakpoint. These settings are applied via a loop in the body selector in _global.scss.

// fonts
$font-primary: 'Nunito', 'Arial', sans-serif;
$font-secondary: 'Poppins', 'Arial', sans-serif;

// base sizes (used before breakpoints are applied)
$base-font-size: 1.6em;
$base-line-height: 1.5;

Hierarchy

All text will default to the global $base-font-size unless otherwise specified. Font size hierarchy is defined in _global.scss and can be implemented by extending the %font-size placeholder via @extend %font-size{$size}. This is predominantly used in Headings but can be used with any element.

// Font Sizes
%font-size{
    &--xsmall { font-size: 0.6em}
    &--small { font-size: 0.8em }
    &--normal { font-size: 1em}
    &--medium { font-size: 1.5em}
    &--large { font-size: 1.75em}
    &--xlarge { font-size: 2em}
    &--xxlarge { font-size: 2.5em}
    &--xxxlarge { font-size: 3em}
}

Links

Text links are styled via _global.scss with a simple hover animation applied. The smooth(color) mixin is used to apply consistent transition to the text color - see mixins for more information. By default, colour is applied via global color() map settings and should be considered in all projects. See colour settings for details. The grid system does not currently support default button styles.

a {
    @include smooth(color);
    color: color(primary);
    text-decoration: underline;
    &:visited {
        color: color(secondary);
        text-decoration: none;
    }
    &:hover {
        color: color(tertiary);
    }    
    &:focus, &:active {
        color: color(tertiary);
    }
}

Images

As defined in _global.scss, all img elements have height and width set to auto, and max-width set to 100%. This prevents images from breaking out of their containers, and also allows width/height to scale as appropriate when a more specific dimension is defined.

Example: the image below has an actual size of 300px x 200px however the fixed size of the container (200px x 200px) forces the image to scale down to fit.

Lists

Support for basic lists is included for ul, ol and dl. Minimal styling is included, with lists styled via _global.scss. The placeholder class %lists is used to output consistent styling to both ul and ol lists:

%lists {
    margin: .5em 0 1.25em 2.5em;
    list-style-position: outside;
    ul, ol {
        margin: 0 0 0 2em;
    }
}

Nested lists are also supported, with additional styling to indent the left margin for visual separation.

Utilities

Hiding elements

Elements can be hidden via .hide to hide them on all devices, or by using breakpoint selectors to target specific screen sizes - see breakpoint settings. The grid system will automatically generate the classes as required, and name them according to the class names defined in the $breakpoints map in _settings.scss. An element can also be set to be hidden (but still take up space on the page) with the class .invisible. These css classes can be chained to hide/show elements at different breakpoints.

.invisible This text is invisible but still takes up space on the page. So if it falls across multiple lines, the table cell will grow to fit it.
.hide This text will be hidden on all devices.
.hide-below-sm This text will be hidden below the sm breakpoint.
.hide-above-sm This text will be hidden above the sm breakpoint.
.hide-below-md This text will be hidden below the md breakpoint.
.hide-above-md This text will be hidden above the md breakpoint.
.hide-below-lg This text will be hidden below the lg breakpoint.
.hide-above-lg This text will be hidden above the lg breakpoint.
.hide-below-sm .hide-above-lg This text will only be shown in the md breakpoint.

Floats

Floats are defined in _global.scss and allow an element to be floated to the left or right using .alignleft and .alignright respectively. !important is used to avoid specificity issues.

Floats can be cleared by adding .clearfix to the parent container, or by extending the class via @extend .clearfix

.alignnone

This div does not float, and appears where it is in the flow of the document.

.alignleft

This div is floated to the left.

.alignright

This div is floated to the right.

Floating images

Images, like all elements, can be floated by applying these utility classes. Additional styling automatically applies margins to create white space alongside text. These styles are defined in _global.scss.

The %embedded-content placeholder variable will center an image and clear floats below a certain breakpoint (default sm) to prevent tiny columns of text.

img {
    &.alignleft {
        @extend %embedded-content;
        @include rem(margin, 0 15px 15px 0);
    }
    &.alignright {
        @extend %embedded-content;
        @include rem(margin, 0 0 15px 15px);
    }
    &.alignnone {
        @extend %embedded-content;
        @include rem(margin, 15px 0);
    }
    &.aligncenter {
        @extend %embedded-content;
        display: block;
        @include rem(margin, 15px auto);
    }
}

%embedded-content {
    @include below(sm) {
        float:none !important;
        @include rem(margin, 10px auto);
        display: block;
    }
}

Grid

Grid Settings

At the heart of this framework is a grid that is completely customisable with the following settings:

  • Columns
  • Push/Pull include y/n
  • Offset include y/n
  • Debug mode on/off
  • Endless breakpoints with customisable names
  • For each user-defined breakpoint:
    • Container width
    • Container gutter
    • Column gutter
    • Base font size
    • Base line height

Column amount

The underlying benefit of Jellyfish is it's flexible approach to grid layout. You can define the precise amount of columns required, and on compile, the scss logic will produce all of the required css styling. So whether you want a 4 column, 7 column, 12 column or 3000 column grid (Not sure why you would want this, but knock yourself out...) Jellyfish has you covered. For the purposes of this documentation, we are using a 12 column grid - but feel free to go wild with a 5 column grid if your project requires it.

This ability to define only the required amount of columns allows the compiled code to be optimised and lean, reducing unnecessary bloat.

Column quantity isrequired and is defined by the variable $cols in _settings.scss and the logic for generating the grid is defined in _grid.scss.

Push / Pull

Columns can be pushed (to the right) or pulled (to the left) by a certain number of columns, if $has_push_pull is set to true in _settings.scss. .push and .pull classes are generated automatically by _grid.scss

Elements can be pushed/pulled at different breakpoints as defined in the breakpoint settings by using the classes .push-$breakpoint-$cols and .pull-$breakpoint-$cols respectively. Push and pull can also be reset at a breakpoint with the classes .push-$breakpoint-0 and .pull-$breakpoint-0 respectively.

.push and .pull classes can be chained eg. .push-sm-2 .push-md-5 .push-lg-0

.push-sm-6 .push-md-8 .push-lg-4
.pull-sm-6 .pull-md-0. pull-lg-4
.pull-md-8 .pull-lg-4

Grid Structure

Jellyfish uses a simple grid structure, consisting of a .container, .row and several .col elements. These elements are defined in _grid.scss

.containerBy default, the grid will fill all available space (ie 100% width). To restrict the width of the grid, a .container should be used. The container is centered and can be customised at each breakpoint with the following settings: width, max-width, padding (left/right). See grid breakpoint settings for information.
.rowHouses a number of .col elements and is automatically cleared with .clearfix to prevent flow issues. A .row can be used on the same element in conjunction with a .container. To preserve equal margins on the left and right, the only elements inside a .row should be .col elements
.colA single vertical column. Can be used with breakpoint based modifiers as below to change size at different breakpoints. With no modifiers, a .col element will be 100% width.

Settings

Debug mode

Jellyfish comes with a handy debug mode that can be enabled in two ways:

  1. by setting $debug-mode-on to true in _settings.scss.
  2. by adding the class .jf-debug to the body element (either within the code itself or through a browser inspector).

Debug mode allows you to quickly see the structure of the grid, and diagnose any layout issues.

When debug mode is enabled, all .row elements will glow in pink and all .col elements will glow in turquoise.

Additionally, all .row elements will have their background striped to show the number of columns in the grid structure.

Providing that $has_push_pull and/or $has_offset are set to true in _settings.scss, any element affected by a .push, .pull or .offset at the active breakpoint will be highlighted with a red border; the side with the heavier stroke indicating the direction to which the action has occured.
